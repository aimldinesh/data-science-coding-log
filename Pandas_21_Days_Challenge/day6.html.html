Fix the revenue column
Easy

You receive a sales DataFrame where the revenue column loaded as object dtype because values contain currency symbols and commas (e.g. "$1,200.50"). Some rows also contain the string "N/A".

Write code to: (1) convert revenue to a proper float64 column, turning invalid entries into NaN, (2) fill NaN values with the median, (3) verify the dtype is now float64.

INPUT DATA

order_idrevenueregion
01001$1,200.50North
11002$850.00South
21003N/AEast
31004$3,400.75North
41005$920.00West
revenue (after)
01200.50

33400.75

Strip $ and , using str.replace with regex, then use pd.to_numeric(errors='coerce') to handle the "N/A". Remember: median fill after conversion, not before.
SOLUTION
# Step 1 — strip currency characters
df['revenue'] = df['revenue'].str.replace(r'[$,]', '', regex=True)

# Step 2 — convert to numeric, N/A → NaN
df['revenue'] = pd.to_numeric(df['revenue'], errors='coerce')

# Step 3 — fill NaN with median
df['revenue'] = df['revenue'].fillna(df['revenue'].median())

# Verify
print(df['revenue'].dtype)   # float64
print(df['revenue'].isnull().sum())  # 0

Clean and deduplicate a user table
Medium

A user table has multiple issues: inconsistent name casing, extra whitespace, a duplicate user (same email, different timestamp), and an age column that loaded as object because one row contains "unknown".

Write a clean pipeline that: (1) standardises names to title case with no extra spaces, (2) fixes the age column to int64 (drop rows where age is unparseable), (3) deduplicates by email keeping the most recent record, (4) resets the index after deduplication.

INPUT DATA

nameemailagesignup_ts
0alice smithalice@x.com282023-01-10
1 BOB JONES bob@x.comunknown2023-03-05
2Carol Whitecarol@x.com342023-07-20
3bob jonesbob@x.com352023-11-01
Order matters: (1) clean strings first, (2) fix age with pd.to_numeric(errors='coerce') then dropna then astype(int), (3) sort by signup_ts before deduplicating with keep='last'.
SOLUTION
# Step 1 — clean names
df['name'] = df['name'].str.strip().str.title()

# Step 2 — fix age: coerce → drop NaN → cast
df['age'] = pd.to_numeric(df['age'], errors='coerce')
df = df.dropna(subset=['age'])
df['age'] = df['age'].astype('int64')

# Step 3 — sort by timestamp, keep latest per email
df['signup_ts'] = pd.to_datetime(df['signup_ts'])
df = df.sort_values('signup_ts')
df = df.drop_duplicates(subset=['email'], keep='last')

# Step 4 — reset index
df = df.reset_index(drop=True)
print(df)

Missing value audit and fill strategy
Medium

You are given a patient dataset. Write a complete missing value pipeline:

(1) Print a missing value audit: column, null count, null %, dtype — sorted by null % descending.
(2) Create a binary feature bmi_missing before filling bmi.
(3) Fill bmi using group-wise median by gender and smoker combined.
(4) Fill charges with the global median.
(5) Drop the region column (60%+ missing — not recoverable).

Do all fills leakage-safe: split into train/test first, compute statistics on train only.

INPUT SCHEMA

age      int64      — no nulls
gender   object     — no nulls  (male / female)
bmi      float64    — 18% null
smoker   object     — no nulls  (yes / no)
region   object     — 62% null
charges  float64    — 5% null
Split with df.sample(frac=0.8) first. Then for the group-wise bmi fill: compute group medians on train using groupby().transform(). For test, you'll need to merge the group medians back in. Creating bmi_missing before any fill is critical — do it on the full df before splitting.
SOLUTION
# Step 1 — audit
audit = pd.DataFrame({
    'nulls'  : df.isnull().sum(),
    'pct'    : (df.isnull().mean()*100).round(1),
    'dtype'  : df.dtypes
}).sort_values('pct', ascending=False)
print(audit[audit['nulls']>0])

# Step 2 — binary indicator (before any fill, on full df)
df['bmi_missing'] = df['bmi'].isnull().astype(int)

# Step 3 — drop high-missing column
df = df.drop(columns=['region'])

# Step 4 — train/test split
train = df.sample(frac=0.8, random_state=42)
test  = df.drop(train.index)

# Step 5 — group-wise bmi fill (train stats only)
bmi_medians = train.groupby(['gender','smoker'])['bmi'].median()

def fill_bmi(df_, medians):
    def filler(row):
        if pd.isnull(row['bmi']):
            return medians.get((row['gender'], row['smoker']),
                                df_['bmi'].median())
        return row['bmi']
    df_['bmi'] = df_.apply(filler, axis=1)
    return df_

train = fill_bmi(train.copy(), bmi_medians)
test  = fill_bmi(test.copy(),  bmi_medians)

# Step 6 — charges fill (train median only)
charges_med = train['charges'].median()
train['charges'] = train['charges'].fillna(charges_med)
test['charges']  = test['charges'].fillna(charges_med)

Rewrite the slow pipeline
Hard

The code below works correctly but is painfully slow on large DataFrames. It uses Python loops and apply where vectorised operations would be 10–100x faster.

Rewrite the entire pipeline using only vectorised operations (np.where, np.select, .str accessor, direct arithmetic). No loops, no apply. Verify your output matches the original.

SLOW CODE TO REWRITE

# This works but is very slow — rewrite it
results = []
for idx, row in df.iterrows():
    name  = row['name'].strip().lower()
    sal   = row['salary']

    # tax rate
    if   sal < 30000: tax = sal * 0.05
    elif sal < 80000: tax = sal * 0.15
    else:             tax = sal * 0.30

    # seniority flag
    senior = 1 if sal > 80000 else 0

    # net pay
    net = sal - tax

    results.append({
        'name_clean': name,
        'tax'       : tax,
        'senior'    : senior,
        'net_pay'   : net
    })

result_df = pd.DataFrame(results)
Four operations: (1) .str.strip().str.lower() for name, (2) np.select with conditions list for the three-band tax, (3) (df['salary'] > 80000).astype(int) for the binary flag, (4) direct subtraction for net_pay. Build the result as a new DataFrame using pd.DataFrame({...}).
SOLUTION
# All vectorised — no loops, no apply

# 1. Clean names
name_clean = df['name'].str.strip().str.lower()

# 2. Tax with np.select (3 bands)
conditions = [
    df['salary'] < 30000,
    df['salary'] < 80000
]
rates = [0.05, 0.15]
tax_rate = np.select(conditions, rates, default=0.30)
tax = df['salary'] * tax_rate

# 3. Senior flag — vectorised boolean cast
senior = (df['salary'] > 80000).astype(int)

# 4. Net pay
net_pay = df['salary'] - tax

# 5. Assemble result
result_df = pd.DataFrame({
    'name_clean': name_clean,
    'tax'       : tax,
    'senior'    : senior,
    'net_pay'   : net_pay
})
