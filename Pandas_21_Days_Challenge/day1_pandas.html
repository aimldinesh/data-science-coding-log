What is a Series?

A Series is a 1-dimensional labeled array — think of it as a single column with an index attached to every value.

s = pd.Series([85000, 92000, 78000], index=['Alice','Bob','Carol'])

indexvalue
Alice
85000
Bob
92000
Carol
78000

dtype: int64  |  name: None

Key attributes:

.index — the labels (here: names)

.values — the underlying numpy array

.dtype — data type of values

.name — column name when inside a df

# Three ways to create a Series
s1 = pd.Series([10, 20, 30])                        # default 0,1,2 index
s2 = pd.Series([10, 20, 30], index=['a','b','c'])   # custom index
s3 = pd.Series({'alice': 85000, 'bob': 92000})      # from dict — keys become index

# Accessing values
s2['a']       # → 10  (label-based)
s2[0]        # → 10  (position-based, works but avoid)
s2.loc['a']  # → 10  (explicit label — preferred)
s2.iloc[0]  # → 10  (explicit position — preferred)
Interview tip: interviewers often ask "what's the difference between a Series and a numpy array?" — the key answer is the labeled index. Operations align on index labels, not position.

What is a DataFrame?

A DataFrame is a 2-dimensional labeled table — a dict of Series sharing the same index. Every column is a Series.

df = pd.DataFrame({'name':['Alice','Bob','Carol'], 'salary':[85000,92000,78000], 'dept':['Eng','Eng','HR']})

namesalarydept
0Alice85000Eng
1Bob92000Eng
2Carol78000HR

Key attributes:

.shape — (rows, cols) tuple

.columns — column labels

.index — row labels

.dtypes — dtype per column

.info() — full summary

.describe() — stats summary

# Creating DataFrames
df = pd.DataFrame({'name': ['Alice','Bob'],
                    'salary': [85000, 92000]})

# From a list of dicts — very common in real code
records = [{'name':'Alice', 'salary':85000},
           {'name':'Bob',   'salary':92000}]
df = pd.DataFrame(records)

# Quick inspection
df.shape        # → (2, 2)
df.columns      # → Index(['name','salary'])
df.dtypes       # → name: object, salary: int64
df.info()       # memory usage, non-null counts, dtypes
df.describe()   # mean, std, min, max for numeric cols
concept

loc vs iloc — the most tested topic

This single distinction trips up more candidates than anything else. Know it cold.

.loc — label-based

Selects by index label and column name. Both ends of a slice are inclusive.

# rows by label
df.loc[0]
df.loc[0:2]   # includes row 2!
df.loc[[0,2]]

# row + column
df.loc[0, 'name']
df.loc[:, 'salary']
df.loc[0:2, ['name','dept']]

.iloc — position-based

Selects by integer position. End of slice is exclusive (like Python lists).

# rows by position
df.iloc[0]
df.iloc[0:2]  # excludes row 2
df.iloc[[0,2]]

# row + column
df.iloc[0, 1]
df.iloc[:, 1]
df.iloc[0:2, 0:2]
Common trap: when the index has integer labels (0, 1, 2...) loc and iloc look the same — until you reset or reindex. Use .loc for labels, .iloc for positions, always. Never rely on bare df[0] for rows. 
# Boolean masking — the most common real-world pattern
df[df['salary'] > 80000]
df[(df['dept'] == 'Eng') & (df['salary'] > 80000)]

# .loc with boolean mask (preferred for assignments)
df.loc[df['salary'] > 80000, 'level'] = 'senior'

# .at and .iat — single cell, fastest
df.at[0, 'name']   # label-based single value
df.iat[0, 1]       # position-based single value

Quick quiz — 3 questions

Answer from memory before looking at the code. This is exactly how interviews work.

Q1. You have a DataFrame with a non-default string index. Which selects the row labelled "alice"?

Correct! .loc uses label-based indexing. .iloc only accepts integers. df['alice'] selects a column named 'alice', not a row.
Not quite. .loc is for labels, .iloc only accepts integer positions. df['col'] selects a column, not a row.

Q2. df.loc[0:3] on a default-indexed DataFrame returns how many rows?

Correct! .loc slices are inclusive on both ends — rows 0, 1, 2, and 3 are all returned. This is unlike Python list slicing which excludes the end.
.loc slices are inclusive on both ends. So 0:3 returns rows 0, 1, 2, AND 3 — 4 rows total. This catches many people.

Q3. What does df['salary'] return?

Correct! df['col'] returns a Series — a 1D labeled array. To keep it as a DataFrame use df[['col']] (double brackets).
df['col'] returns a Series. To get a single-column DataFrame use double brackets: df[['col']].

The SettingWithCopyWarning trap

One of the most common bugs in pandas — and a very popular interview question.

# BAD — chained indexing. May silently NOT modify df.
df[df['dept'] == 'Eng']['salary'] = 100000   # ⚠ SettingWithCopyWarning

# Why? Two separate __getitem__ calls create a copy on the first step.
# The assignment goes to the copy, not the original df.
The first bracket creates a temporary copy. The second bracket writes to that copy — and your original DataFrame is unchanged. Pandas may or may not warn you.
# GOOD — always use .loc for any assignment
df.loc[df['dept'] == 'Eng', 'salary'] = 100000   # ✓ correct

# GOOD — if you intentionally want a copy to work on
subset = df[df['dept'] == 'Eng'].copy()
subset['salary'] = 100000   # ✓ fine — explicit copy
Rule of thumb: if you're reading data, chaining is fine. If you're writing/assigning, always use .loc.

Reading — fine to chain

df[df['dept']=='Eng']['name']
df.query("salary > 80000")['dept']

Writing — use .loc

df.loc[df['dept']=='Eng', 'bonus'] = 5000
df.loc[0, 'name'] = 'Alice Smith'
