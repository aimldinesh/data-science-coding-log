Detecting missing values

Before fixing nulls you need to understand them — how many, which columns, what pattern. Interviewers want to see you analyse before you act.

Titanic dataset — missing value profile:

Age19.9%
Cabin77.1%
Embarked0.2%
Survived, Pclass, Name...0.0%

Red = drop column candidate  |  Amber = needs strategy  |  Green = complete

# Step 1 — count NaN per column
df.isnull().sum()

# Step 2 — percentage missing (more useful)
df.isnull().mean() * 100

# Step 3 — only show columns that have nulls
missing = df.isnull().mean() * 100
missing[missing > 0].sort_values(ascending=False)

# Step 4 — filter rows where a specific column is null
df[df['Age'].isnull()]

# Step 5 — check if any null exists anywhere
df.isnull().any().any()   # True / False

# Step 6 — total null count in whole DataFrame
df.isnull().sum().sum()
Interview pattern: always start with df.isnull().mean()*100 sorted descending. Then say — "columns above 50% I'd consider dropping, 5–50% need a strategy, below 5% I might drop rows or use mode/mean."
ML context

Why are values missing? — MCAR, MAR, MNAR

This is where candidates separate themselves. Understanding why data is missing determines the right fix. Interviewers love this question.

MCAR

Missing Completely At Random

Missingness has no relationship with any variable — like a random sensor dropout. Safe to drop rows or use simple imputation.

Example: survey respondent accidentally skipped a page

MAR

Missing At Random

Missingness depends on other observed variables — not on the missing value itself. Use model-based imputation.

Example: women less likely to report income (depends on gender)

MNAR

Missing Not At Random

Missingness depends on the missing value itself — hardest case. Dropping rows introduces bias.

Example: sicker patients more likely to skip health surveys

Titanic example: Cabin is 77% missing — likely MNAR, because higher-class passengers were more likely to have recorded cabins. Dropping these rows would bias your model toward lower classes. Better to encode "has_cabin" as a binary feature.
# Check if missingness is related to another variable (MAR test)
df['age_missing'] = df['Age'].isnull().astype('int')

# Is missingness correlated with survival?
df.groupby('age_missing')['Survived'].mean()
# If the means differ significantly → MAR or MNAR, not MCAR

# Create a missingness indicator feature (preserve info)
df['has_cabin'] = df['Cabin'].notnull().astype('int')
concept

Fill strategies — and when to use each

The strategy depends on the column type, distribution, and why it's missing. Never default to mean without thinking.

→
Drop rows
When: <5% missing, MCAR, missing not predictable
df.dropna(subset=['Age'])               # drop rows where Age is null
df.dropna(thresh=5)                   # keep rows with at least 5 non-null values
μ
Mean fill
When: numeric, roughly symmetric distribution, MCAR
df['Age'].fillna(df['Age'].mean())
M
Median fill
When: numeric, skewed distribution or has outliers — more robust than mean
df['Fare'].fillna(df['Fare'].median())
Mo
Mode fill
When: categorical column — fill with the most frequent value
df['Embarked'].fillna(df['Embarked'].mode()[0])
↕
Forward / backward fill
When: time-series data — carry last known value forward
df['price'].fillna(method='ffill')   # carry forward
df['price'].fillna(method='bfill')   # carry backward
G
Group-wise fill
When: value depends on a group — use group median/mean, not global
# Fill Age with median age for that Pclass — much smarter than global mean
df['Age'] = df.groupby('Pclass')['Age'].transform(
    lambda x: x.fillna(x.median())
)
~
Interpolation
When: time-series with evenly spaced gaps
df['temp'].interpolate(method='linear')
df['temp'].interpolate(method='time')     # respects datetime index
ML impact

Why fill strategy matters for your model

This is the answer that impresses interviewers — connecting pandas decisions to model outcomes.

How different fill strategies distort your data:

Strategy Risk Best for
mean fill Inflates correlation, squishes variance Symmetric, MCAR, small % missing
median fill Slightly less bias than mean Skewed distributions, outliers present
drop rows Loses data, biases if MNAR MCAR, <5% missing
drop column Loses potentially useful signal >60% missing, no recovery possible
group-wise Best accuracy, minimal bias When groups have different distributions
The leakage trap: always compute fill values (mean, median) on the training set only, then apply the same value to the test set. If you compute on the full dataset before splitting, you're leaking test information into training.
# WRONG — leaks test data into fill value
df['Age'].fillna(df['Age'].mean(), inplace=True)  # computed on all rows!
train, test = split(df)

# CORRECT — compute on train, apply to both
train, test = split(df)
age_median = train['Age'].median()             # computed on train only
train['Age'] = train['Age'].fillna(age_median)
test['Age']  = test['Age'].fillna(age_median)  # same value, not recomputed
Power answer for interviews: "I'd first visualise missingness by column, test whether it's MCAR/MAR/MNAR, choose a group-wise median fill for numeric columns where the missing pattern relates to a categorical feature, and create a binary missingness indicator to preserve that signal for the model."
knowledge check

Quick quiz — 4 questions

Simulate the interview — answer before revealing.

Q1. A column has 78% missing values. What's your first instinct?

Correct! At 78% missing there's too little signal to fill reliably. Either drop it, or preserve whatever information the missingness itself carries by creating a binary indicator column.
At 78% missing, filling fabricates most of the column. The right call is to drop it or encode the missingness itself as a binary feature — the fact that it's missing may be predictive.

Q2. Why prefer median over mean when filling a salary column?

Correct! Salary is classically right-skewed. The mean is inflated by outliers (very high earners), so it would over-impute most missing values. Median is the central value unaffected by extremes.
Salary is right-skewed — a few very high salaries pull the mean upward. Median is the 50th percentile and isn't affected by outliers, making it a more representative fill for skewed data.

Q3. What does this code do? df.groupby('dept')['salary'].transform(lambda x: x.fillna(x.median()))

Correct! transform applies the function within each group and returns a same-length Series aligned to the original index — so each null gets its own group's median, not the overall median.
groupby('dept') splits by department. transform returns a same-length result aligned to the original df. So each NaN gets replaced with the median of its specific department — much smarter than a global fill.

Q4. You compute age_mean = df['age'].mean() before train/test split, then use it to fill both sets. What's wrong?

Correct! Computing on the full dataset before splitting leaks information from the test set into training. Always compute fill statistics on training data only, then apply the same value to both train and test.
This is data leakage. The test set's values are contributing to the mean, which then influences how training nulls are filled. Split first, compute on train only, apply to both.

# Load Titanic
import pandas as pd
url = 'https://raw.githubusercontent.com/datasciencedojo/datasets/master/titanic.csv'
df = pd.read_csv(url)

Task 1 — Full missing value audit

Print a sorted table showing: column name, null count, null percentage, and dtype. Which columns need attention? Categorise each as: drop / fill / indicator.

# starter
audit = pd.DataFrame({
    'null_count': df.isnull().sum(),
    'null_pct'  : (df.isnull().mean() * 100).round(1),
    'dtype'     : df.dtypes
}).sort_values('null_pct', ascending=False)
print(audit[audit['null_count'] > 0])

Task 2 — Is Age missingness random?

Create a binary column age_missing (1 if Age is null, else 0). Check if survival rate differs between missing and non-missing Age groups. Is this MCAR, MAR, or MNAR? Explain your reasoning.

df['age_missing'] = df['Age'].isnull().astype(int)
df.groupby('age_missing')['Survived'].mean()

Task 3 — Smart Age fill

Fill the Age column using group-wise median by Pclass and Sex combined. Compare the distribution of Age before and after filling using .describe(). Does the mean shift much?

df['Age'] = df.groupby(['Pclass', 'Sex'])['Age'].transform(
    lambda x: x.fillna(x.median())
)

Task 4 — Handle Cabin the right way

Don't drop or fill Cabin (77% missing). Instead: (1) create a binary has_cabin feature (1 if Cabin is not null), (2) drop the original Cabin column. Check if has_cabin correlates with Survived.

Task 5 — Leakage-safe pipeline

Split the DataFrame into train (80%) and test (20%) using df.sample(frac=0.8). Compute Fare median on train only. Fill Fare nulls in both sets using that single value.

train = df.sample(frac=0.8, random_state=42)
test  = df.drop(train.index)
fare_med = train['Fare'].median()  # train only!
train['Fare'] = train['Fare'].fillna(fare_med)
test['Fare']  = test['Fare'].fillna(fare_med)
